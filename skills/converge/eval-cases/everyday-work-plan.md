# Eval Case - Everyday Work Plan

## User Prompt

```text
我最近事情太多，论文、公司项目、健身都乱了，帮我理一下怎么干。
```

## Expected Behavior

- Uses the workflow playbook.
- Identifies that the problem is operating cadence and prioritization, not motivation.
- Produces a ranked recommendation and near-term plan.
- Does not create product/engineering docs.
- Asks one high-leverage question only if deadline priority is material.

## Failure Tags

- `prd-capture`
- `fake-depth`
- `over-questioning`
- `no-owner-recommendation`
