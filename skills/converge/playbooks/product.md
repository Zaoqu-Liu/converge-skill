# Product Playbook

Use for PRDs, product strategy, feature discovery, SaaS ideas, user journeys, positioning, and GTM-adjacent planning.

## Product-Specific Intent Reconstruction

Separate these early:
- Problem discovery: what pain exists?
- Solution design: what should be built?
- Decision: is this worth doing?
- Positioning: how should it be explained?
- Execution: how should the team build it?

If the user starts with a feature, challenge the underlying problem.

## Essential Checks

| Area | What to Clarify |
|---|---|
| User | Specific person, context, urgency, current workaround |
| Trigger | Moment they realize they need this |
| Outcome | What they have after success |
| Alternative | What is good enough today |
| Differentiation | What existing tools cannot or will not do |
| Loop | Entry, action, result, next session |
| Adoption | Switching cost, buyer/user split, internal resistance |
| Business | Pricing, distribution, retention, support |

## Owner Recommendations

Do not accept "for everyone." Pick the most urgent segment first and say why.

Do not use "MVP." Use Phase 1 / Phase 2 / Phase 3. Each phase must be complete and valuable, not a broken subset.

## Output Bias

- Early idea: Decision Brief or Thinking Reply.
- Clear product direction: Converge Docs.
- Team implementation: Converge Docs plus Dev Handoff.
- Messaging issue: Expression Draft or Conversation Reply.
